設計初衷與核心目標

為什麼我們需要這個工作區套件？解決 AI 與人類協作時的痛點。

領域知識結構化

AI 常缺乏特定專案的領域知識。本套件設計了自建知識庫（Brain），讓 AI 在動手前先載入並理解流程規範與品質標準，避免寫出不符合要求的程式碼。

安全邊界與隱私

嚴防敏感資訊與 API 金鑰洩漏。透過隱私安全合規設計（Privacy Audit），定義明確的隱私邊界，確保整個工作區可安全進行 Git 版本控制與公開分享。

合約化開發流程

以 Working Contract 規範 AI 的邊界與行為。使用 slash commands（如 /game-production）引導 AI，確保開發過程有計畫、有追蹤、有交接。

全局架構總覽 (四層架構)

從外部套件層到核心工作區治理，建立清晰的職責分工。

1. 套件層 (Package Layer) 對外描述
README.md / MANIFEST.json / PRIVACY-AUDIT.md — 套件說明與分享檢查
2. 腳本層 (Scripts Layer) 自動化工具
install / verify-install / privacy-scan / pack.ps1 — 系統自動化腳本
3. 範例層 (Examples Layer) 實作參考
game-mini-demo / visual-mini-demo — 虛構專案範例，用以展示工作流
4. 工作區模板層 (Workspace Template Layer) 核心內容
AGENTS.md / workflows / brain / planned / shared — 複製到工作區的實體核心

分層架構的優勢

透過分層，我們能讓使用者與 AI 快速定位所需資源：

  • 極簡複製： 實際使用時僅需複製 workspace-template。
  • 工具鏈支持： 外層的 scripts 提供自動化安裝與隱私安全掃描。
  • 自給自足： 專案解壓縮後，AI 即可從 AGENTS.md 入門並自給自足。

套件層與自動化腳本

管理套件生命週期、安裝程序與發布前的安全掃描。

套件管理文件

  • 00.README.md

    套件的對外說明書。包含手動與自動安裝指南，以及主要的工作流程說明。

  • 01.file_mapping.md

    全局檔案映射表，描述每個目錄和檔案的作用。建議引導 AI 優先讀取。

  • PRIVACY-AUDIT.md

    隱私審查機制，列出發布前必跑的 checklist，防止私人或公司專案資料外洩。

PowerShell 自動化腳本

  • install.ps1

    將 workspace-template 複製安裝到使用者指定的目標工作區資料夾。

  • verify-install.ps1

    驗證目標工作區是否成功安裝所有必要檔案（如 AGENTS.md 等）。

  • privacy-scan.ps1

    靜態分析程式碼，掃描高風險字串（API keys、本機路徑、私人 URLs）。

  • pack.ps1

    一鍵驗證、執行隱私掃描，並自動打包壓縮成乾淨無敏感資訊的發布包。

工作區模板層 (核心治理與合約)

一旦複製進工作區，AI 與人類便透過這些文件達成協作共識。

AGENTS.md (治理憲法)

工作區的最高指南。當 AI 首次進入工作區時，必須指示其優先閱讀此檔案。

  • 定義專案目錄結構規範與各目錄職責
  • 訂立敏感資訊不可寫入規則與隱私防護邊界
  • 宣告主要的工作流程與 slash commands 入口

工作流程合約 (workflows/)

定義特定的指令合約，規範 AI 在執行該指令時的輸入、輸出與限制。

  • /game-production ─ 規範遊戲企劃發想、關卡切片、資產 Manifest
  • /visual-design ─ 規範視覺方向定義、HTML 原型開發、視覺 QA
  • /science-education ─ 自然領域數位教材與闖關遊戲開發合約
  • /linebot-ai ─ [NEW] LINE Bot 整合 OpenRouter AI 與 Sheets 教學合約
  • /codex-handoff ─ 定義任務狀態（Task State）日誌與 AI 交接格式

Brain 自建知識庫運作機制

引導 AI 在執行任務前先「讀書」，載入對應領域的標準流程與護欄。

1. 路由引導 (SKILL.md)

AI 的入口地圖。根據當前任務類型（如遊戲設計、網頁視覺），將 AI 的注意力精確導向對應的 Skill 目錄，避免無效的全局檢索。

2. 領域 Skill 定義 (skills/*/SKILL.md)

定義該領域的核心工作流（Core Flow）、標準產出（Outputs）與品質護欄（Guardrails），作為 AI 行為的合約標準。

3. 深層參考文獻 (references/)

收納更為具體的操作細節，如：HTML 原生 CSS 布局規則、精靈圖資產清單 JSON 格式、或特定測試流程的 Checklist。

AI 知識系統對照

本套件的 Brain、Codex 原生規則與 Antigravity 記憶系統有何不同？

定位：專案級「領域教科書」

將人類團隊的領域知識結構化與模組化，是專為特定任務（如遊戲製作、視覺包裝）設計的知識載體。

  • 作用範圍： 僅在目前的工作區資料夾內生效。
  • 版本控制： ✅ 可完全納入 Git 版本追蹤與團隊共享。
  • 攜帶性： ✅ 隨資料夾複製，打包即可轉交給其他協作者。

運作核心機制

透過兩層式的路由（入口引導至深入參考文件），避免 AI 面臨 context window 溢出或龐大文件混淆，達到極高效率的讀書機制。

定位：專案級「工作區憲法」

定義這個專案工作區的基本目錄結構、執行原則與隱私保護防護邊界，為最基本的規章制度。

  • 作用範圍： 僅在目前的工作區資料夾內生效。
  • 結構複雜度： 扁平化的單一文件，定義基礎規則。
  • 撰寫主體： 由人類開發者手動編寫與管理。

侷限性

主要用作行為治理（Governance），例如哪些路徑不可提交。無法用來教育 AI 如何實作具體複雜的業務邏輯或開發管線。

定位：系統級「AI 個人隨身筆記」

AI 在對話中自動累積、產生的知識快取，如 API 呼叫模式、重構紀錄、歷史除錯經驗。

  • 作用範圍： 全域生效（存於本機 AppData 目錄）。
  • 可攜性： ❌ 無法跟著專案資料夾一起用 Git 共享。
  • 生命週期： 隨著使用時間自動演進，但可能逐漸過時。

三者互補關係

AGENTS.md 負責「治理」，brain/ 提供「專業教科書」，KI 則是 AI 的「經驗筆記」。三者完美互補，讓 AI 具備規則、知識與實戰經驗。

推薦 AI 協作開發工作流

一套結構化、漸進式的 AI 協作開發流程，最大化人機開發效能。

STEP 1
計畫撰寫

利用設計簡報模板，對齊設計意圖並拆解任務

STEP 2
資料庫選擇

優先選用 SQLite 或原生 localStorage，避免過度設計

STEP 3
型別合約

定義明確 of Schema/Interface，預防重構崩潰

STEP 4
軟體設計

模組化小檔案，介面先行，並立即撰寫驗證測試

STEP 5
代碼美工

CSS/SVG 優先，外部素材以 Asset Manifest 統一管理

自然領域專案開發流程

專為自然學科設計的數位教材與闖關遊戲開發管線。

STEP 1
下載教材與課綱

下載課本 PDF 及課綱資料，作為知識源頭

STEP 2
課程架構分析

AI 分析課程並於指定位置產出分析架構文件

STEP 3
教材與遊戲設計

根據課程架構設計簡報、闖關遊戲與評量規準

STEP 4
Google Sheets + GAS

學生資料優先用 Google Sheets 與 Apps Script 儲存

STEP 5
發布至 GitHub

無指定則預設將教材與遊戲專案發布至 GitHub 託管

課程架構文件產出位置： G:\學科教材\01_自然5上課本PDF\textbook_analysis_report.md
當 AI 讀取課本與課綱後，會依據上述路徑儲存課程架構。另外本實作計畫書亦同步儲存於 G:\vibeCode_backup\planned\implementation_plan.md 以供追蹤。

如何快速開始？

準備好你的工作區，讓 AI agent 發揮 100% 的威力。

快速啟動 3 步驟

  • 1 建立與複製： 建立新目錄，將 workspace-template 內的所有內容複製進去。
  • 2 引導 AI 閱讀： 在對話框中輸入：
    「請先讀取 AGENTS.md 與 brain/SKILL.md 理解工作區治理規則與架構。」
  • 3 觸發工作流： 使用合約指令（如 /game-production, /science-education 或 /linebot-ai）開始開發專案！

工作區維護守則

  • 定期執行 privacy-scan.ps1 以防金鑰上傳。
  • 完成大階段後，請使用 /codex-handoff 自動產生交接日誌。
  • 新的公共知識應記錄至 brain/skills/ 累積經驗。
